Design

Learn How to Program Makeover (3-Day Project)

Redesign this site you're reading right now; LearnHowToProgram.com! Do you think a more classic "textbook" feel would be more functional and appealing? What about something more modern and sleek? How would you make it easier to navigate? Should it be more richly-featured? More minimal?

Steps

Here is the workflow you will follow to adhere to best practices of The Web Design Process, Mobile-First Design, and Design Thinking simultaneously:

  1. Target User Personas and Stories. Take time and care at this step! As a student at Epicodus, you're in the target audience for this site. This puts you in a unique position to better understand users' needs, and how they use the interface. But don't forget to consider other types of users too! Identify the application's target audience, and all user personas. Record a full list of user stories in your README before advancing.
  2. Sketches: Using your user stories as a guide, begin sketching. Create mobile versions of each planned view first, following mobile-first design. Experiment, try different approaches and layouts, discuss thoroughly with your partner, and tweak and rework as necessary. Remember, you should have a fairly finalized layout on paper before wireframing. When you've completed mobile sketches, move on to their desktop equivalents. Then add photographs of paper sketches you're using in the final project to your project repository and README.
  3. Wireframes: After both mobile and desktop sketches are finalized, create higher-fidelity digital versions of each in Sketch software. Collaborate with your partner to make decisions about interface details that weren't finalized in the sketches (colors, fonts, etc.) Add both mobile and desktop wireframes to your project repository and README.
  4. Prototypes: Then you'll create both mobile and desktop prototypes of your design. Consider how the application will move and respond. Consider navigation; how does . Tweak and experiment as necessary, until you have a prototype you feel confident with. But don't worry about this yet, we'll cover prototyping in Wednesday's homework!
  5. Coding: Finalized prototype in hand, you can begin coding an actual site that reflects the design created in your sketches, wireframes, and prototypes!
  6. Further Exploration: If you complete all steps above before the end of Thursday, restart steps for Admin areas described below. Remember, user stories and personas for admins of the site will likely differ from standard users.
